linux 报告了个BUG并解决了

By ousui on 2016-07-17

https://bugzilla.redhat.com/show_bug.cgi?id=1357235
https://github.com/hluk/CopyQ/issues/530

copyq 是一款跨平台的高级的剪贴板管理工具。
十分好用，定制功能强大，在 linux 平台找了很久才找到，qt gui，很好看

然而 fedora 官方源装完以后，无法选择语言……明明自己编译的就可以。
经过一晚上折腾，不断的编译尝试最后还看了 rpmbuild，最终确认问题是 cmake 编译的时候有个参数写错了。。导致了查找 translations 路径有问题。
所以就造成了，如果在 /usr 目录下执行命令就没问题，但是在别的目录下执行就会找不到的奇怪问题。

这一晚上折腾的，学了好多只是。。

最后再安利下 copyq 吧，真的很好用：

  • 可以插件，可以扩展
  • 可以执行 ecmascript，可以执行 shell
  • 全平台
  • 支持图片复制，不支持文件复制，但是复制文件以后，会把路径存储
  • 支持多标签
  • 可以自定义多种操作，例如转换文字为二维码；过滤一些内容到单独的标签（例如图像）
  • 好处多多，自行体会。
